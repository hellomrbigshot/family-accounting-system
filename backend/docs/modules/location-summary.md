# 消费地址功能需求总结

## 功能概述

在支出系统中添加消费地址记录功能，支持记录每次消费的地点，并可以根据地址查看消费历史和统计信息。

## 核心需求

### 1. 地址记录
- ✅ 在创建/编辑支出时可以记录消费地址
- ✅ 如果未指定地址，自动使用当前时间段的默认地址
- ✅ 地址信息存储在支出记录中
- ✅ 支持记录消费人员（多成员场景，可选）

### 2. 默认地址管理
- ✅ 支持设置默认地址
- ✅ 默认地址可以按时间段设置（每段时间可以不一致）
- ✅ 时间段不能重叠
- ✅ 支持编辑和删除默认地址
- ✅ 支持查看当前生效的默认地址

### 3. 地址查询
- ✅ 可以根据地址查询消费记录
- ✅ 支持地址关键词搜索
- ✅ 支持按日期范围筛选

### 4. 地址统计
- ✅ 统计去过的所有地址
- ✅ 显示每个地址的消费次数和总金额
- ✅ 显示首次和最后访问日期
- ✅ 支持按成员筛选和分组统计（多成员场景）

### 5. 时间轴查看
- ✅ 按时间顺序查看消费地址
- ✅ 显示每天去过的地址
- ✅ 显示该地址的消费次数和总金额
- ✅ 支持查看详细消费记录
- ✅ 支持按成员筛选时间轴（多成员场景）

## 数据模型

### Expense 模型扩展
- 新增 `location` 字段（可选，字符串，最大 200 字符）
- 新增 `member` 字段（可选，字符串，最大 50 字符，用于多成员场景）

### UserDefaultLocation 模型（新建）
- `userId`: 用户ID
- `startDate`: 开始日期
- `endDate`: 结束日期（可选，null表示永久有效）
- `location`: 默认地址
- `isActive`: 是否激活

## API 接口

### 默认地址管理
- `POST /user-default-locations` - 创建默认地址
- `GET /user-default-locations` - 获取默认地址列表
- `GET /user-default-locations/current` - 获取当前默认地址
- `PUT /user-default-locations/:id` - 更新默认地址
- `DELETE /user-default-locations/:id` - 删除默认地址

### 支出 API 扩展
- `POST /expenses` - 创建支出（支持 location 字段）
- `PUT /expenses/:id` - 更新支出（支持 location 字段）
- `GET /expenses` - 查询支出（支持按 location 筛选）

### 地址查询统计
- `GET /expenses/locations` - 获取消费地址列表
- `GET /expenses/locations/timeline` - 获取地址时间轴
- `GET /expenses/locations/:location` - 根据地址查询消费记录

## 注意事项

1. **数据迁移**：现有支出记录的 `location` 和 `member` 字段为 `null`，不影响现有功能
2. **默认地址**：如果未设置默认地址，支出记录的 `location` 为 `null`
3. **时间段管理**：系统会检测时间段重叠，确保同一时间段只有一个激活的默认地址
4. **性能考虑**：为地址和成员字段创建索引，优化查询性能
5. **多成员场景**：`member` 字段是可选的，支持单用户和多成员两种使用方式
6. **成员名称**：成员名称是自由文本，建议使用常用成员列表减少输入错误
